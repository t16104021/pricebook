# LINE 半自動查價設計

## 目標

讓管理者把客戶傳到個人 LINE 的詢價內容，整理成固定指令後傳給 LINE 官方帳號。系統從現有 Supabase 價格資料查詢結果，再由官方帳號把內部完整資料回覆給管理者。客戶仍與管理者的個人 LINE 聯絡，不直接操作查價系統。

## 使用流程

1. 客戶傳送詢價訊息到管理者的個人 LINE。
2. 管理者傳送 `查價 客戶名稱 產品編號` 給 LINE 官方帳號。
3. LINE 將訊息事件送至 Supabase Edge Function。
4. Edge Function 驗證 LINE Webhook 簽章及管理者的 LINE User ID。
5. Edge Function 解析指令，讀取指定 Pricebook 帳號的 `pricebook_data.payload`。
6. 系統查找客戶、產品及最新價格紀錄。
7. LINE 官方帳號將完整內部資料回覆給管理者。
8. 管理者確認內容後，自行回覆客戶。

## 系統架構

### LINE 官方帳號

- 啟用 Messaging API 與 Webhook。
- 接收管理者傳送的文字指令。
- 使用 Reply API 將查詢結果回覆到同一個個人 LINE 對話。

### Supabase Edge Function

- 作為 LINE Webhook 的公開 HTTPS 端點。
- 以原始 request body 和 LINE Channel Secret 驗證 `x-line-signature`。
- 只處理文字訊息。
- 比對允許查價的 LINE User ID；未授權使用者不得取得任何價格資料。
- 解析查價指令、查詢資料並組合回覆。
- 使用伺服器端環境變數保存 LINE Channel Secret、Channel Access Token、允許的 LINE User ID 與 Pricebook 擁有者 ID。

### 現有 Supabase 資料

- 沿用 `public.pricebook_data`，不複製產品或價格資料。
- 使用固定的 Pricebook 擁有者 ID 取得正確帳號的資料列。
- 從 `payload.products` 查詢產品定價、客戶售價及歷史紀錄。
- 查價結果以日期最新的一筆作為目前價格。

## 指令格式

標準格式：

```text
查價 客戶名稱 產品編號
```

範例：

```text
查價 長青商行 ABC-100
```

解析規則：

- 忽略指令前後多餘空白。
- 關鍵字必須是 `查價`。
- 客戶名稱必須完整符合，不自動猜測相近客戶。
- 產品優先以 SKU 完整符合。
- 沒有完整符合時，以 SKU 或產品名稱進行不分大小寫的部分搜尋。
- 客戶名稱或產品搜尋文字含空白時，使用雙引號包住，例如 `查價 "Great North Co" "ABC 100"`。

## 查詢結果

### 單一產品

回覆內部完整資料：

```text
客戶：長青商行
產品：ABC-100 高效濾芯
產品定價：NT$1,200
定價日期：2026/06/01
客戶售價：NT$980
售價日期：2026/06/27
備註：年度合約價
```

金額使用千分位；缺少備註時顯示 `無`。

### 多個產品

部分搜尋找到多筆時，不自行選擇，最多列出前 10 筆：

```text
找到 3 個產品：
1. ABC-100 高效濾芯
2. ABC-200 標準濾芯
3. ABC-300 工業濾芯

請輸入完整產品編號。
```

超過 10 筆時加上提示，要求輸入更多產品編號字元縮小範圍。

## 錯誤處理

- 格式不正確：回覆正確格式與一個範例。
- 客戶不存在：回覆 `查無客戶：{名稱}`。
- 產品不存在：回覆 `查無產品：{搜尋文字}`。
- 尚無產品定價：該欄顯示 `尚未設定`。
- 該客戶尚無售價：仍顯示產品定價，客戶售價欄顯示 `尚未設定`。
- 非授權 LINE User ID：回覆 `此帳號沒有查價權限`，不透露資料是否存在。
- 資料庫或 LINE API 暫時失敗：回覆簡短失敗訊息並在 Edge Function 記錄技術錯誤，不在 LINE 顯示金鑰或內部細節。
- 重複 Webhook：以 LINE Webhook Event ID 去重，避免重複回覆。

## 安全性

- 每個 Webhook 請求都必須驗證 LINE 簽章。
- Channel Secret、Access Token 與 Supabase Service Role Key 只放在 Edge Function Secrets。
- 前端及 GitHub 儲存庫不得包含上述機密。
- 查詢前同時驗證 LINE User ID 與固定 Pricebook 擁有者 ID。
- 回覆只傳回管理者發起的對話，不主動推送給其他帳號。

## 不包含範圍

- 不讀取個人 LINE 的私人聊天。
- 不自動把結果貼回客戶的個人 LINE。
- 不讓客戶直接查詢價格。
- 不在第一版加入自然語言或 AI 判斷。
- 不修改現有產品管理、Excel 匯入匯出及登入流程。

## 測試

- 正確指令能取得完整產品與客戶價格。
- SKU 完整符合優先於部分符合。
- 部分搜尋一筆、多筆、超過 10 筆及零筆結果。
- 不同大小寫、額外空白及引號內容解析。
- 客戶不存在、尚無定價、尚無客戶售價。
- 非授權 LINE User ID 無法取得資料。
- 錯誤或缺少簽章的 Webhook 被拒絕。
- LINE Webhook 重送不會造成重複回覆。
- Supabase 或 LINE API 失敗時回覆安全且可理解的訊息。

## 完成標準

- 管理者能用個人 LINE 傳送固定查價指令給官方帳號。
- 正確查詢會在同一對話收到內部完整價格資料。
- 模糊產品搜尋會列出候選項目，不會回覆錯誤產品。
- 除指定管理者外，其他 LINE 使用者無法取得任何價格資料。
- 整個服務部署在雲端，管理者的電腦關機時仍能使用。
